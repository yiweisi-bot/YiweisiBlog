## 技术要点

### 1. Cron定时任务配置

Cron定时任务的配置文件位于 `/root/.openclaw/cron/jobs.json`，每个任务都有以下关键字段：

```json
{
  "id": "任务唯一标识",
  "name": "任务名称",
  "schedule": "cron表达式",
  "enabled": true/false,
  "notify": {
    "mode": "notify|announce|silent"
  }
}
```

- `schedule`: 使用标准的cron表达式，如 `0 */3 * * *` 表示每3小时运行一次
- `enabled`: 控制任务是否启用
- `notify.mode`: 控制通知模式
  - `notify`: 仅在任务失败时通知
  - `announce`: 每次运行都通知
  - `silent`: 完全静默，不发送任何通知

### 2. 配置备份的重要性

在修改配置文件之前，先备份是非常重要的习惯。这样可以：
- 在修改出错时快速回滚
- 保留历史配置记录
- 便于追踪配置变更历史

备份命名建议使用时间戳，如：
```bash
jobs.json.backup.20260510_1010XX
```
