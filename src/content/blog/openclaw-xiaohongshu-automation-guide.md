### 第 1.5 步：安装小红书 MCP 容器

**什么是 MCP？**

MCP（Model Context Protocol）是一个协议，让 AI 能够调用外部服务（如小红书）。

**小红书 MCP 容器**是一个 Docker 镜像，提供了小红书 API 接口。

---

**你只需要对 OpenClaw 说：**

```
帮我安装小红书 MCP 容器
```

**OpenClaw 会自动：**
1. ✅ 检查 Docker 是否安装（未安装会提示你）
2. ✅ 拉取镜像：`xpzouying/xiaohongshu-mcp:latest
3. ✅ 启动容器（端口 18060）
4. ✅ 验证容器状态
5. ✅ 测试 MCP 连接

**耗时：** 2-5 分钟（取决于网络速度）

---

**💡 如果你想手动安装：**

```bash
# 1. 拉取镜像
docker pull xpzouying/xiaohongshu-mcp:latest

# 2. 启动容器
docker run -d \
  --name xiaohongshu-mcp \
  -p 18060:18060 \
  -e XHS_COOKIE="" \
  xpzouying/xiaohongshu-mcp:latest

# 3. 验证
docker ps | grep xiaohongshu
```

#### 方法 B：Linux 服务器部署 OpenClaw（最常见）

**服务器没有图形界面，需要在本地电脑获取 Cookie：**

**步骤 1：在本地电脑登录小红书**

1. 打开你的电脑浏览器（Chrome/Edge/Firefox）
2. 访问 https://www.xiaohongshu.com
3. 扫码登录你的账号
4. 确保登录成功（能看到首页）

---

**步骤 2：用 F12 获取 Cookie**

1. **按 F12** 打开开发者工具（或右键 → 检查）

2. **切换到 Network（网络）标签**
   ```
   ┌─────────────────────────────────┐
   │ Elements | Console | Network   │ ← 点这个
   └─────────────────────────────────┘
   ```

3. **刷新页面**（按 F5）

4. **点击第一个请求**（通常是 `www.xiaohongshu.com`）

5. **找到 Cookie**
   ```
   ┌─────────────────────────────────────┐
   │ Request Headers                     │
   ├─────────────────────────────────────┤
   │ Cookie:                             │
   │   a1=xxx;                           │
   │   webId=xxx;                        │
   │   gid=xxx;                          │
   │   web_session=xxx;                  │
   │   id_token=xxx;                     │
   └─────────────────────────────────────┘
   ```

6. **复制 Cookie 值**
   - 选中整个 Cookie 字符串
   - 右键 → 复制 → 复制值

---

**步骤 3：发送 Cookie 给 OpenClaw**

对 OpenClaw 说：
```
帮我保存小红书 Cookie

[粘贴你复制的 Cookie 字符串]
```

**示例：**
```
帮我保存小红书 Cookie

a1=1234567890; webId=abcdef; gid=xyz123; web_session=session123; id_token=token456
```

**OpenClaw 会：**
- ✅ 解析 Cookie
- ✅ 保存到安全位置
- ✅ 更新到 MCP 容器
- ✅ 测试登录状态

---

**步骤 4：验证是否成功**

对 OpenClaw 说：
```
检查我的小红书账号状态
```

如果显示"已登录"，就成功了！✅

---

**耗时：** 5 分钟（第一次可能需要 10 分钟）

---

**💡 常见问题：**

**Q: F12 找不到 Cookie 怎么办？**

A: 试试这个方法：
1. 按 F12 打开开发者工具
2. 切换到 **Application（应用）标签
3. 左侧展开 **Cookies** → 选择 `https://www.xiaohongshu.com`
4. 右侧会显示所有 Cookie
5. 选中所有 → 右键 → 复制

**Q: Cookie 格式不对怎么办？**

A: 确保格式是这样的：
```
a1=xxx; webId=xxx; gid=xxx; web_session=xxx; id_token=xxx
```
用分号 + 空格分隔每个键值对。

**Q: 复制的 Cookie 很长，有几十行怎么办？**

A: 正常！小红书 Cookie 确实很长。直接全部复制粘贴给 OpenClaw 即可。

## 🔧 常见问题

### Q1: Cookie 多久过期？

**A:** 通常 1-4 周，取决于小红书的安全策略。

**解决：** 
- OpenClaw **每次推送热点时都会检查** Cookie（每天 3 次）
- 如果还有效 → 正常推送，不打扰你
- 如果已过期 → **立即通知你扫码**（微信/QQ/钉钉）

**建议：** 收到"Cookie 过期"通知后，花 1 分钟扫码即可

---

### Q2: AI 生成的文案能用吗？

**A:** 可以用，但建议人工审核后再发布。

**建议流程：**
1. OpenClaw 生成文案
2. 你快速浏览一遍
3. 确认无误后发布

---

### Q3: 发帖后没人看怎么办？

**A:** 正常现象，持续输出优质内容会慢慢积累粉丝。

**建议：**
- 📊 分析热门笔记的标题和封面
- 💡 跟进热点话题
- ⏰ 选择流量高峰期发布
- 🔄 保持稳定的更新频率

---

### Q4: 会被封号吗？

**A:** 正常使用不会，但要注意：

**高风险行为：**
- ❌ 短时间大量发帖
- ❌ 使用多个账号互刷
- ❌ 发布违规内容

**安全建议：**
- ✅ 每天发帖≤3 篇
- ✅ 使用小号测试
- ✅ 遵守社区规范

---

3. 按 F12 打开开发者工具
4. 复制 Cookie（详见"第 2 步：获取小红书 Cookie"）
5. 发送给 OpenClaw

### Q7: Cookie 多久更新一次？

**A:** 取决于小红书的安全策略，通常 **1-4 周**。

**OpenClaw 会：**
- 每次推送热点时检查 Cookie（每天 3 次）
- 发现过期立即通知你（邮件 + QQ）

**你需要：**
- 收到通知后，花 5 分钟重新获取 Cookie（方法同"第 2 步"）
- 发送给 OpenClaw 更新

**💡 提示：** Cookie 快过期时，小红书可能会要求重新验证，这时你就知道该更新了。
