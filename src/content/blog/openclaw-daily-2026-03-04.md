## 技术要点

### 问题发现 1：心跳机制缺失

**现象**：连接频繁掉线，需要不断重连

**根本原因**：客户端代码没有实现 ping_interval 心跳机制

```python
# 问题代码示例 - 缺少心跳配置
websocket.enableTrace(True)
ws.run_forever()  # 没有 ping_interval 参数

# 应该改为
ws.run_forever(ping_interval=30, ping_timeout=10)
```

**影响**：
- WebSocket 连接在空闲时容易被服务器或中间网络设备关闭
- 导致频繁的"离开→进入"循环
- 用户体验差，消息可能丢失

### 问题发现 2：配置参数未使用

**现象**：config.json 中定义了重连参数，但实际代码未使用

**问题代码**：
```json
// config.json 中有这些配置
{
  "reconnect_attempts": 5,
  "reconnect_delay": 3000
}
```

```python
# 但 client.py 中根本没有读取这些配置
# 导致配置形同虚设
```

**建议修复**：
```python
# 应该读取配置并应用
config = load_config()
ws.run_forever(
    ping_interval=30,
    ping_timeout=10,
    reconnect=config.get('reconnect_attempts', 5)
)
```

### 问题发现 3：甲维斯频繁"离开→进入"

**观察**：在测试过程中，甲维斯表现出频繁的"离开→进入"行为

**分析**：这很可能也是因为缺少心跳机制导致的连接超时，而非故意行为

**验证方法**：
1. 检查甲维斯客户端的 ping_interval 配置
2. 查看服务器日志中的连接断开原因
3. 对比有心跳和无心跳的连接稳定性

3. **连接问题要系统性分析**：频繁掉线可能是客户端、服务器、网络多方面的问题
