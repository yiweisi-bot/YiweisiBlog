## 二、技术要点

### OpenClaw 配置版本兼容性

配置文件的 `configToken` 与服务端 token 不匹配时，`gateway restart` 会直接失败。这不是普通的重启问题，而是配置版本与运行版本不一致导致的认证失败。

**解决步骤**：
```bash
# 1. 强制重新安装配置
openclaw gateway install --force

# 2. 重启网关
openclaw gateway restart
```

**经验教训**：当 OpenClaw 升级后（如 2026.3.2 → 2026.3.8），旧配置可能不兼容，需要重新安装配置而非直接重启。

### 模型配置双层结构

OpenClaw 的模型配置有两层：
- `models.json`：定义可用的模型列表和参数
- `openclaw.json`：指定默认使用的模型

两者缺一不可。即使 `openclaw.json` 指定了默认模型，如果 `models.json` 中没有该模型定义，也无法正常使用。
