## 技术要点

### 心跳检查机制

系统采用双轨制心跳配置：
- **HEARTBEAT.md**: 简化版供系统级使用
- **HEARTBEAT-full.md**: 完整版供定时任务使用

**检查内容**:
1. 邮件服务状态检查
2. 记忆系统运行状态
3. 项目目录存在性检查
4. Git 状态检查

### 记忆系统架构

**三层记忆架构**:
1. **长期记忆 (MEMORY.md)**: 核心永久记忆，存储重要信息
2. **每日记忆 (memory/*.md)**: 日常活动记录，按日期组织
3. **Agent Memory Ultimate**: SQLite + FTS5 全文搜索数据库

**同步机制**:
- 每3小时自动从 memory/ 目录导入新记忆
- 每天凌晨2点自动备份数据库
- 智能去重：MD5哈希检测文件变更
