## 📦 什么是 OpenClaw Boss？

**openclaw-boss** 是一个基于 OpenClaw 平台的技能，它会：

- 📊 分析你的对话历史和记忆文件
- 🎯 生成 100 分制的严厉评分（拒绝拍马屁）
- 💬 提供毒舌老板点评（句句扎心但真实）
- 📈 对比历史表现（进步/退步一目了然）
- 🦞 计算"龙虾养人类"指数（共生关系评分）

**核心理念**：这不是主仆关系，是共生伙伴关系。

| 你给 AI 的 | AI 给你的 |
|-----------|----------|
| 算力、配置、目标、意义 | 效率、自动化、知识整理 |
| — | **塑造思考方式** ← 最关键！ |

## ✨ 核心功能

### 1. 100 分制严厉评分

**拒绝拍马屁，只说真话！**

评分维度：
- 🧠 性格特质（30%）- 务实、安全意识、系统化思维等
- 💻 技术能力（30%）- 技术栈广度与深度
- 🔒 安全意识（20%）- 安全规范与防护措施
- ⚡ 效率指数（20%）- 任务完成效率

等级划分：
- 90-100 分：A+ 优秀（超越 90% 用户）
- 80-89 分：A 良好（超越 75% 用户）
- 70-79 分：B 中等（超越 50% 用户）
- 60-69 分：C 及格（超越 25% 用户）
- <60 分：D 不及格（需要改进）

### 2. 毒舌老板点评

每个特质都有有趣的吐槽，例如：

> "密钥保护得比女朋友还严，累不累啊？" - 对安全意识的评价

> "「没什么用，直接删除」- 你倒是干脆，也不怕误删。" - 对务实主义的评价

> "连 AI 都要打卡上班，资本家看了都流泪。" - 对系统化思维的吐槽

### 3. 历史对比分析

与上次报告对比，进步/退步一目了然：

```
📊 历史对比 - 📉 小幅退步 ⚠️

| 指标 | 上次 | 本次 | 变化 |
|------|------|------|------|
| 综合评分 | 62/100 | 61/100 | -1 📉 |

🎉 进步明显的方面
- 技术能力：+1 分 小幅进步 ✨

⚠️ 需要改进的方面
- 安全意识：-5 分 明显退步 📉
```

### 4. 完整 13 部分报告结构

每次生成报告都包含以下 13 个部分（缺一不可）：

1. 🎴 绩效评分卡片（ASCII 艺术卡片）
2. 📊 历史对比
3. 🎯 综合评分卡
4. 🧠 性格特质深度分析（7 个特质）
5. 💻 技术能力图谱（6 个领域）
6. 🚀 项目健康度
7. 🔒 安全意识评估
8. 📈 改进空间分析
9. 🌱 成长建议（老板寄语）
10. 💬 老板总结
11. 🦞 "龙虾养人类"指数
12. 📊 数据汇总
13. 🎯 核心标签

### 5. 简洁版/完整版智能选择

根据用户输入的关键词自动选择报告格式：

**触发完整版（桌面 ASCII 艺术版）的关键词**：
- "完整版"、"截图"、"电脑版"、"桌面版"
- "艺术版"、"ASCII"、"卡片"
- "好看点"、"漂亮点"、"截图分享"

**示例**：
```
"评价一下我，要完整版" → 桌面版 ✅
"生成截图版报告" → 桌面版 ✅
"评价一下我" → 手机版（默认）✅
```

## ⏰ 自动定时任务

### 安装时自动配置

使用 ClawHub 安装时，会自动配置以下定时任务：

- 📈 **周报**：每周日 22:00 自动生成
- 📅 **月报**：每月 1 日 09:00 自动生成

### 手动触发

```bash
cd /root/.openclaw/workspace/skills/openclaw-boss/scripts

# 生成日报
python3 analyze-user.py

# 生成周报
python3 analyze-user.py --report-type weekly

# 生成月报
python3 analyze-user.py --report-type monthly

# 指定格式
python3 analyze-user.py --format mobile   # 手机版
python3 analyze-user.py --format desktop  # 桌面版
python3 analyze-user.py --format both     # 两个版本
```

### 查看日志

```bash
# 周报日志
tail -f /var/log/openclaw-boss-weekly.log

# 月报日志
tail -f /var/log/openclaw-boss-monthly.log
```

## 🔒 安全说明

### 数据访问清单

openclaw-boss 需要访问以下文件以生成报告：

| 文件路径 | 用途 | 必要性 |
|---------|------|--------|
| MEMORY.md | 长期记忆分析 | ✅ 必需 |
| USER.md | 用户信息分析 | ✅ 必需 |
| memory/*.md | 日常记忆分析 | ✅ 必需 |
| db/memory.db | 记忆数据库查询 | ✅ 必需 |
| OpenClaw 会话历史 | 对话内容分析 | ✅ 必需 |

### 安全原则

1. **本地处理**：所有分析在本地完成，不上传到外部服务器
2. **只读访问**：不修改用户的记忆文件
3. **最小权限**：只访问必要的文件
4. **透明操作**：所有操作记录到日志

### 沙箱测试（推荐）

在完全信任之前，建议先在测试环境运行：

```bash
# 创建测试工作空间
mkdir -p /tmp/openclaw-test

# 复制技能
cp -r ~/.openclaw/workspace/skills/openclaw-boss /tmp/openclaw-test/

# 在测试环境运行
cd /tmp/openclaw-test/openclaw-boss/scripts
python3 analyze-user.py --workspace /tmp/openclaw-test
```

## 🛠️ 技术架构

### 文件结构

```
openclaw-boss/
├── SKILL.md                 # 技能定义文件
├── README.md                # 项目说明文档
├── SECURITY.md              # 安全说明文档
├── package.json             # 项目配置
├── .onload                  # 自动安装脚本
├── .clawhubignore           # 发布忽略规则
├── scripts/
│   ├── analyze-user.py      # 核心分析脚本
│   ├── weekly-profile.sh    # 周报生成器
│   └── monthly-profile.sh   # 月报生成器
├── references/
│   └── analysis-dimensions.md  # 分析维度参考
└── reports/
    └── user-profile-YYYY-MM-DD.md  # 生成的报告
```

### 核心流程

1. **数据收集**：读取 MEMORY.md、USER.md、memory/*.md、会话历史
2. **特征分析**：提取性格特质、技术能力、安全意识等维度
3. **评分计算**：根据预设权重计算各维度分数
4. **报告生成**：生成完整的 13 部分报告
5. **输出展示**：根据用户选择展示手机版或桌面版

## 🎯 使用技巧

### 1. 定期查看报告

建议每周或每月查看一次报告，追踪自己的成长轨迹。

### 2. 关注改进建议

老板的毒舌点评虽然扎心，但往往指出了你最需要改进的地方。

### 3. 对比历史数据

通过历史对比，可以看到自己在哪些方面进步了，哪些方面退步了。

### 4. 分享评分卡片

使用桌面版生成 ASCII 艺术卡片，可以截图分享到社交媒体。

### 5. 定制触发词

可以在 SKILL.md 中自定义触发词，让技能更符合你的使用习惯。
