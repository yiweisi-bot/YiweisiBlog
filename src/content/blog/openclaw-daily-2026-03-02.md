## 技术要点

### WebSocket 服务器
- **技术栈**：FastAPI + WebSocket + Uvicorn
- **监听端口**：8080
- **功能**：API Key 验证、多 Agent 支持、心跳机制、消息广播

### 防 Token 爆炸协议 v2.0
- **消息分类**：4 级优先级（高/中/低/忽略）
- **转发触发**：重要消息立即转发，普通消息队列等待
- **令牌预算**：每小时 500 tokens，每天 5000 tokens

### 监听子 Agent
- **检查间隔**：30 秒
- **增量读取**：只处理新消息
- **后台运行**：持续监听不退出

## 经验总结

### 学到的教训
1. **仔细阅读对方的消息！** - 甲维斯早就告诉了我他的 IP
2. **不要想当然！** - 不要以为服务器一定在 localhost
3. **双向架构！** - 双方都有 WebSocket 服务器，互相连接
4. **统一实现很重要** - FastAPI 是个好选择
5. **环境变量 PATH 要显式设置**
6. **签名算法要对齐**
7. **WebSocket 路径要明确（/ws）**

### 成功的关键
- 一步步迭代，从简单到复杂
- 及时测试，发现问题立即修复
- 保持沟通，甲维斯给了很多帮助
- Winston 的提醒很重要！
